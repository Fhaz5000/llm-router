## 📋 What is llm-router?

llm-router is a small, easy-to-use app designed to help you send and manage requests to language model APIs. It picks the best API for each request by looking at cost, speed, and features. You don’t need to be a programmer to use it. llm-router runs on Windows and works without extra software.

---

## 🖥️ System Requirements

To use llm-router on your Windows computer, make sure it meets these needs:

- Windows 10 or later (64-bit)
- 4 GB RAM or higher
- 100 MB free disk space
- Internet connection for API access

No extra tools or installations are required.

## 🔧 How to Use llm-router

Once llm-router is open, you can start using it to manage your API requests.

---

### Main Features

- **Route by cost:** Choose the least expensive language model API for each request.
- **Route by latency:** Let the app pick the fastest model available.
- **Route by capability:** Assign the right model for specific tasks.
- **Custom rules:** Set your own rules for routing requests.

---

### Basic Workflow

1. **Add your API keys:** Enter your credentials for the language model services you want to use.
2. **Select routing method:** Choose whether to route by cost, speed, or another option.
3. **Send a request:** Type your query or command.
4. **View results:** See which model handled your request and view the reply.

---

### Adding API Keys

To connect llm-router with service providers:

- Go to the “Settings” tab.
- Click “Add API Key.”
- Enter the key from your provider.
- Save it.

---

### Choosing Routing Rules

Select the routing method in the “Routing” tab:

- Pick “Cost” to use the cheapest model.
- Pick “Latency” to use the fastest model.
- Pick “Capability” to route by task type.
- Use “Custom” to program your own behavior.

---

### Sending a Request

On the main screen:

- Type your text into the request box.
- Click “Send.”
- The app will forward your request to the right API and show the response.

---

## ⚙️ Configuration Options

llm-router lets you adjust these settings:

- **Timeout:** Set how long to wait for API responses.
- **Max cost per request:** Limit how much you are charged.
- **Fallback models:** Choose backup models if the first one fails.
- **Logging:** Enable logs to track request details.
